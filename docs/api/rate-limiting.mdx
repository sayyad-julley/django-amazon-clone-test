## Overview

Our API implements a robust, flexible rate limiting mechanism to protect against abuse and ensure service stability. The rate limiting is implemented using decorators that can be easily applied to different API endpoints with configurable parameters.

## Rate Limiting Strategies

### Authentication Rate Limiting

#### Strict Authentication Rate Limit
- Applies to login endpoints
- **Limit**: 5 requests per minute
- **Features**:
  - Progressive delay with exponential backoff
  - Prevents brute-force attacks
  - Logs rate limit violations

#### REST Authentication Rate Limit
Dynamically configures rate limits based on authentication action:

1. **Login**:
   - 5 requests per minute
   - Progressive delay
   - Detailed violation tracking

2. **Registration**:
   - 3 requests per 5 minutes
   - Progressive delay
   - Prevents mass account creation

3. **Password Reset**:
   - 2 requests per hour
   - Progressive delay
   - Protects against password reset abuse

### Other Rate Limit Types

- **Standard API Rate Limit**: 10 requests per minute for general API endpoints
- **User Profile Rate Limit**: 10 requests per minute for profile and logout endpoints

## Implementation Details

Rate limiting is implemented using the `enhanced_rate_limit` decorator in `DjangoEcommerceApp/utils/rate_limiter.py`. Key features include:

- IP-based tracking
- Configurable request limits and time windows
- Detailed logging
- Retry-After headers
- Progressive delay mechanism

## Example Usage

```python
@strict_auth_rate_limit
def adminLoginProcess(request):
    # Login logic here
    pass

@rest_auth_rate_limit
def registerUser(request):
    # Registration logic here
    pass
```

## Best Practices

- Apply appropriate rate limiting decorators to sensitive endpoints
- Monitor rate limit logs for potential abuse
- Adjust limits based on traffic patterns and security requirements
