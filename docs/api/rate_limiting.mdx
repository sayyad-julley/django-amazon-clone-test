# API Rate Limiting

## Overview
The application provides a robust rate limiting mechanism for authentication and API endpoints to prevent abuse and ensure fair usage.

## Rate Limiting Strategies

### Authentication Endpoints
- **Strict Rate Limit**: 5 requests per minute
- Progressive delay for repeated violations
- Helps prevent brute-force attacks

### Default API Endpoints
- **Standard Rate Limit**: 10 requests per minute
- Prevents excessive API usage

## Configuration

### IP-Based Tracking
- Uses multi-method IP detection:
  1. X-Forwarded-For header
  2. X-Real-IP header
  3. REMOTE_ADDR fallback

### Customization
Rate limiters can be easily customized using the `enhanced_rate_limit` decorator:

```python
@enhanced_rate_limit(
    max_requests=5,         # Maximum requests
    window_seconds=60,      # Time window in seconds
    key_prefix='auth_api',  # Unique cache key
    progressive_delay=True  # Exponential backoff
)
def login_view(request):
    # View logic
```

## Response Headers
When rate limit is exceeded:
- `Retry-After`: Suggested wait time
- `X-RateLimit-Limit`: Maximum allowed requests
- `X-RateLimit-Remaining`: Remaining requests
- `X-RateLimit-Reset`: Window reset timestamp

## Logging
Rate limit violations and successful checks are logged for monitoring and security analysis.
